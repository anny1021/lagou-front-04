### 简答题答案以及说明

- 一、谈谈你是如何理解 js 异步编程的 eventloop、消息队列都是做什么的，什么是宏任务，什么是微任务？

  - 如何理解 js 异步编程的 eventloop

    - 首先，js 是单线程程序语言，代码的执行顺序是从上到下，按顺序执行
    - 因为某些任务比较耗时间，如果等待这个执行完成，在执行之后的代码，页面会出现卡顿，用户体验差，所有有了 事件循环（eventloop），它将代码分为 立即执行的代码，去别的队列执行，比如：webapi ,执行完成后，通知 执行栈 执行完成，再回收结果。在这过程中，所有的任务，都是有自己的排队队列，即消息队列

  - 消息队列是做什么的？

    - 当我们写代码的时候，有很多行代码，异步执行的代码，但是不论多少，代码顺序，自上而下而下，同步，异步执行的代码，分配到不同的任务队列
      - 同步 -》 立即执行 -》 执行栈中
      - 异步
        - 又分为 宏任务，微任务
        - 同步代码在执行栈中执行完成 -》 先执行在等待的 微任务 -》 在执行等待的宏任务
    - 这样整个，自上而下执行栈，微任务栈，宏任务栈，是消息队列，决定当前执行任务的顺序，按次序执行代码

  - 什么是宏任务，什么是微任务
    - 首先宏任务，微任务都是异步任务，与同步任务不同的是，异步任务仅告诉执行栈，hi~这里有个异步任务，就弹出执行栈了。不同的异步任务，会到对应的宏任务，微任务队列去排队
    - 宏任务：大部分的异步调用都是宏任务
    - 微任务：
      - 在同步任务执行完成后的，调用末尾，直接执行 -- 会插队的异步任务
      - 目前的微任务有以下几个
        - promise - 常用
        - MutationObserver - 我还尚未用过
        - process.nextTick - 我还尚未用过
        - queueMicrotask - 我还尚未用过
